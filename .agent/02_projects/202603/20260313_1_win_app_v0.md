## Dictation State Model

The service must operate on a well-defined state machine from Phase 0 onward:

```
Idle → Recording → Transcribing → ReadyToInject → Idle
                                 → PendingRecovery → Idle
                                 → Error → Idle
```

| State | Meaning |
|-------|---------|
| `Idle` | Waiting for hotkey |
| `Recording` | WASAPI capture active, overlay visible |
| `Transcribing` | Audio sent to provider, awaiting response |
| `ReadyToInject` | Text received, injection ladder executing |
| `PendingRecovery` | Injection failed, transcript stored, awaiting Alt+Shift+Z |
| `Error` | Transient failure (no mic, API unreachable), auto-returns to Idle |

## Phases

### Phase 0 — Bootstrap & Walking Skeleton

De-risk the annoying parts first: WinUI 3 shell, tray integration, hotkey plumbing, app lifetime, and local settings/security direction.

- [ ] Create .NET 8 WinUI 3 solution and project structure
- [ ] Confirm app launches cleanly on target dev machine
- [ ] Add H.NotifyIcon.WinUI and verify tray icon appears
- [ ] Verify app can start hidden / minimize to tray
- [ ] Add minimal HotkeyManager spike and confirm both hotkeys are received
- [ ] Verify both hotkeys unregister cleanly on shutdown
- [ ] Add minimal notification mechanism for placeholder actions
- [ ] Define settings storage location and DPAPI secret storage approach
- [ ] Add minimal transcript state model and storage skeleton
- [ ] Add basic logging policy and gitignored temp/output location
- [ ] Confirm `dotnet restore`, `dotnet build`, and `dotnet run` work from a clean checkout

#### Phase 0 Acceptance Criteria
- [ ] App launches successfully
- [ ] Tray icon is visible and usable
- [ ] Both hotkeys trigger placeholder behavior
- [ ] App exits without leaving hotkeys registered
- [ ] Build/run instructions are accurate
- [ ] Repo is ready for Phase 1 implementation without setup ambiguity

### Phase 1 — Core App Infrastructure

- [ ] Real HotkeyManager with dictation toggle + recovery routing
- [ ] App state machine (Idle / Recording / Transcribing / ReadyToInject / PendingRecovery / Error)
- [ ] TranscriptStore with state tracking (success / pending / failed_transcription)
- [ ] Notification plumbing (toast infrastructure for failure/status messages)
- [ ] Recovery hotkey wired to transcript state (pop pending, re-run injection ladder)

### Phase 2 — Audio Capture

- [ ] WASAPI microphone capture
- [ ] PCM → WAV encoding
- [ ] AudioGuard (25 MB upload-size limit — auto-stop before exceeding)

### Phase 3 — Transcription

- [ ] ITranscriptionClient interface
- [ ] OpenAITranscriptionClient (multipart/form-data upload)
- [ ] LocalAITranscriptionClient
- [ ] Model selection support
- [ ] Secure API key storage via DPAPI

### Phase 4 — Injection

- [ ] ClipboardInjector with clipboard preservation (backup → paste → restore)
- [ ] SendInputInjector (Unicode `KEYEVENTF_UNICODE` fallback)
- [ ] Injection ladder orchestration (clipboard → SendInput → pending)
- [ ] Per-app exclusion list enforcement (check at capture-start and recovery)

### Phase 5 — UI

- [ ] Recording overlay (floating indicator)
- [ ] Settings window (provider, model, hotkeys, language, exclusions)
- [ ] Transcript history panel with copy / re-inject
- [ ] Provider and settings UX

### Phase 6 — Validation

- [ ] End-to-end in: Notepad, Chrome textarea, Slack, Word, VS Code
- [ ] Both providers tested (OpenAI + LocalAI)
- [ ] Failure modes validated (no mic, API down, injection blocked, 25 MB limit)
- [ ] Clipboard preservation verified
- [ ] Recovery path validated (Alt+Shift+Z with pending transcripts)

---

## Notes

- Each phase should be completable independently with testable outcomes.
- Phase 0 exists specifically to prevent setup ambiguity from blocking real feature work.
- Recovery hotkey belongs to core infrastructure (Phase 1), not polish.
- Toast notifications belong to core infrastructure (Phase 1), not UI polish.
- Transcript storage model must exist before the history UI panel (Phase 5).
- This plan will be updated as implementation progresses.
