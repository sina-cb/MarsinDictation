## Goal

Implement the macOS v0 dictation app: a menu bar app that records while the user holds **Control+Option**, transcribes via LocalAI (or OpenAI), and injects text at the cursor.

---

## Dictation State Model

```
Idle → Recording → Transcribing → ReadyToInject → Idle
                                 → PendingRecovery → Idle
                                 → Error → Idle
```

| State | Meaning |
|-------|---------|
| `Idle` | Waiting for hotkey |
| `Recording` | AVAudioEngine capture active, HUD visible |
| `Transcribing` | Audio sent to provider, awaiting response |
| `ReadyToInject` | Text received, injection ladder executing |
| `PendingRecovery` | Injection failed, transcript stored, awaiting ⌘⇧Z |
| `Error` | Transient failure (no mic, API unreachable), auto-returns to Idle |

## Key Implementation Differences from Windows

| Aspect | Windows | macOS |
|--------|---------|-------|
| Hotkey mechanism | WH_KEYBOARD_LL (low-level hook) | `NSEvent` global monitors (`flagsChanged`) |
| Hotkey combo | Ctrl+Win (hold) | Control+Option (hold) |
| Recovery hotkey | Alt+Shift+Z | ⌘⇧Z (Command+Shift+Z) |
| Hotkey permissions | None required | None required (modifier-only via `flagsChanged`) |
| Injection permissions | None | Accessibility (for auto-paste; clipboard fallback without) |
| Audio capture | WASAPI via NAudio | AVAudioEngine |
| Text injection | `SendInput` (Unicode keystrokes) | NSPasteboard + CGEvent Cmd+V (with pasteboard restore) |
| Tray/status | H.NotifyIcon (WPF) | NSStatusBar + NSMenu |
| Toast overlay | WPF StatusWindow | NSPanel RecordingHUD |
| Build system | dotnet build/run | XcodeGen + xcodebuild |
| Config | `.env` + `SettingsManager.cs` | `.env` + `EnvLoader.swift` |

## Notes

- **Accessibility for auto-paste only.** Hotkeys work without Accessibility (modifier-only via `flagsChanged`). Accessibility is only needed for auto-paste (CGEvent Cmd+V). Without it, text copies to clipboard and HUD shows "📋 ⌘V to paste".
- **Signing for persistent Accessibility.** With a proper Apple Development certificate + team ID in `Local.xcconfig`, Accessibility permission persists across rebuilds. With ad-hoc signing, it resets each rebuild.
- **Hold-to-record vs toggle.** Unlike the Windows v0 design doc's suggestion of toggle mode, the macOS implementation uses hold-to-record (walkie-talkie style) for a more natural interaction.
- **Pasteboard preservation.** After injection, the previous clipboard contents are restored after 500ms. A `changeCount` check prevents overwriting if something else modified the clipboard in the meantime.
- **Debug playback.** Audio playback after recording is available via the `-debug-playback` launch argument. Disabled by default.
- **CLI-only workflow.** `python3 devtool/deploy.py mac --build` + `open .app` lets you develop without ever opening Xcode UI.
- Each phase was completable independently with testable outcomes.
- Recovery hotkey belongs to core infrastructure (Phase 1), not polish.
- Toast notifications belong to core infrastructure (Phase 1), not UI polish.
- `deploy.py mac` handles XcodeGen regeneration automatically.
